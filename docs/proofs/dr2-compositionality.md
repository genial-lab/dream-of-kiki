## 0. Executive summary

We prove DR-2 (compositionality of the four dream-phase
primitives) under an explicit precondition that excludes the
empirically falsified class of permutations in which
`RESTRUCTURE` precedes `REPLAY`. On the precondition-satisfying
class, `Op = ⟨replay, downscale, restructure, recombine⟩^+`
forms a non-commutative semigroup with additive budget and
functional-composition semantics, and we show additionally that
`recombine` is *not* decomposable as a finite composition of the
other three primitives under the framework's effect semantics —
the fourth generator is semantically irreducible, which justifies
its inclusion as an independent operation rather than a derived
one (cf. `full-draft.md` §3 preamble, §4.4, §4.5).

A fallback contract DR-2' (strict canonical-order semigroup) is
retained for the P_min / P_equ / P_max runtime profiles that
impose ordering by construction.

## 1. Setup

### 1.1 Operations and states

The four canonical primitives are `replay` (Pillar A,
Walker–Stickgold [@walker2004sleep; @stickgold2005sleep]),
`downscale` (Pillar B, Tononi SHY [@tononi2014sleep]),
`restructure` (Pillar D, Friston FEP [@friston2010free;
@rao1999predictive; @whittington2017approximation]), and
`recombine` (Pillar C, Hobson REM [@hobson2009rem;
@solms2021revising]). Each primitive has the typed signature

```
primitive : State × Budget → State × Output
```

where `State = (W, H, M)` is the triple weights / episodic-buffer
/ hierarchy-topology-metadata, and `Budget` is a
component-additive resource triple `(FLOPs, wall_time, energy_J)`
governed by invariant K1.

### 1.2 The free semigroup Op

`Op` denotes the **free semigroup generated by the 4 primitives**
under composition :

```
Op := ⟨replay, downscale, restructure, recombine⟩^+
```

i.e. all finite non-empty sequences (words) over the four
primitives, composed via sequential application. `Op` is
non-commutative by construction (§5 documents the empirically
relevant non-commutative pairs).

**On "semigroup" versus "monoid".** `Op` has no identity element :
framework §4.3 requires every Dream-Episode to contain a
non-empty operation set, so there is no "no-op" primitive and no
neutral element. The structure is therefore a semigroup, not a
monoid. This resolves a previously inconsistent terminology
between framework-C spec §6.1 ("monoid") and §4.5 of the paper
("semigroup") in favour of "semigroup" throughout.

### 1.3 The effect map

Each word `ω = op_n ∘ ... ∘ op_1 ∈ Op` induces an endofunction
`φ(ω) : State → State` via

```
φ(ω)(s) := effect(op_n, ..., effect(op_1, s))
```

where `effect(op, s)` is the State-component projection of
`op(s, b)` under a runtime budget `b` that satisfies K1. The
effect map `φ : Op → End(State)` is a semigroup homomorphism on
the subset of `Op` we prove DR-2 for (see §2 below).

### 1.4 Prior art and positioning

The framework's algebraic treatment of cognitive operations sits
adjacent to the categorical-compositionality programme in
cognitive science [@phillips2010categorial], published in the
target venue, which argues that cognitive compositionality is a
functor on an appropriate category. Relative to that programme,
we (i) restrict to a free non-commutative semigroup rather than
a Cartesian closed category, (ii) commit to an *executable*
predicate and an empirical Conformance Criterion rather than an
interpretive equivalence, and (iii) develop the theory for
dream-phase consolidation specifically, not general cognition. We
borrow notation and methodology from standard free-semigroup
theory (Howie 1995 ; Lothaire 1983/1997 ; see §10 on bibliography
additions) and from the categorical account of deep learning
[@gavranovic2024fundamental].

## 2. Theorem (DR-2, weakened with precondition, 2026-04-21)

**Statement.** For every permutation
`π = (op_0, ..., op_{n-1}) ∈ Op` of length `n ≥ 1` that satisfies
the precondition

```
[P]   ¬∃ i < j : π_i = RESTRUCTURE ∧ π_j = REPLAY,
```

the following three conjuncts hold :

```
(Closure)              π ∈ Op                                    (C)
(Budget additivity)    budget(π) = Σ_{k=0}^{n-1} budget(π_k)     (B)
(Functional composition) φ(π)(s) = effect(π_{n-1}, ...,
                                   effect(π_0, s))               (F)
```

Additionally :

```
(Associativity)  ∀ π_a, π_b, π_c :
   (π_c ∘ π_b) ∘ π_a = π_c ∘ (π_b ∘ π_a)                         (A)
```

(associativity is independent of [P] : it is a property of
function composition in `Set`.)

**Proof structure.** §3 proves (C) under [P]. §4 proves (B). §5
proves (F). §6 proves (A). §7 establishes generator
irreducibility of `recombine` (not part of DR-2 per se, but
required by the framework's free-semigroup commitment in
`full-draft.md` §4.4). §8 documents the non-commutativity
landscape. §9 states the fallback DR-2'.

**Scope of [P].** Three caveats, in order of increasing
implication :

1. *Substrate-dependent* — the falsifying class was identified on
   the real-weight MLX substrate of `kiki_oniric` with canonical
   input shape `(2, 4)` ; on substrates where `restructure` does
   not alter the forward-callable signature of the network, [P]
   may be vacuous. See
   `docs/specs/amendments/2026-04-21-dr2-empirical-falsification.md` §2.
2. *Input-schema-dependent* — the specific failure (MLX `addmm`
   raising `ValueError`) binds to the `(2, 4)` schema ; different
   input shapes may surface different failure loci or none.
3. *Closure-failure-mode* — when [P] is violated, the failure is
   *closure*, not budget additivity and not effect chaining ;
   i.e. the composition cannot be constructed at all rather than
   constructed-but-wrong. A reviewer should not read [P] as
   fixing a numerical error but as carving out the subset on
   which the composition is even typed.

## 3. Closure under precondition [P] (proves (C))

**Claim.** For every permutation `π` satisfying [P], every
prefix `π_{0..k}` admits a well-typed forward pass on the
substrate, i.e. `π ∈ Op`.

**Typing argument.** Each primitive has the signature
`State × Budget → State × Output`. Composition of typed
endomorphisms on `State` is valid by the standard category-theory
closure of typed functions with matching domain and codomain
[@phillips2010categorial; Pierce 2002, ch. 6]. On a substrate
where `restructure` may mutate the callable signature of the
network (e.g. MLX `addmm` on `(2, 4)` input as per §2 caveat 1),
the *operational* type is refined : `REPLAY` additionally
requires the forward-callable signature at runtime. Precondition
[P] is precisely the largest predicate on permutations under
which this refined type is preserved for every prefix.

**Formal composition.** For any two adjacent operations `op_k`
and `op_{k+1}` in a precondition-satisfying `π` :

```
(op_{k+1} ∘ op_k)(s, b)
  = op_{k+1}(s', b - b_k)
    where (s', o_k) = op_k(s, b_k)
```

with `b_k ≤ b` a sub-budget allocation (K1). This recovers the
v0.1 binary-composition form as a special case of the n-ary
statement in §2.

**Empirical evidence.** The Hypothesis-based harness at
`tests/conformance/axioms/test_dr2_compositionality_empirical.py`
exhaustively tests the 24 permutations of the four primitives on
the MLX substrate. The 12 permutations satisfying [P] all pass
the closure test (typed forward call succeeds on the canonical
input). The 12 permutations violating [P] all raise `ValueError`
during the `REPLAY` step, confirming (empirically) that [P] is
both necessary and sufficient for closure on this substrate. On a
substrate that leaves the forward signature invariant under
`RESTRUCTURE`, the failing class collapses to the empty set and
[P] becomes vacuous (consistent with caveat 1 in §2).

**Note on budget terminology.** `budget(op)` denotes the *static
cost* of `op` (resource view, used in the additivity claim
below). The `b` parameter above is the *runtime budget*
(consumption view). The two are related by the constraint
`budget(op) ≤ b_op ≤ b` (static declaration ≤ allocation ≤ DE
budget cap).

**Conclusion 3.** (C) holds for every `π` satisfying [P]. ∎

## 4. Budget additivity (proves (B))

**Claim.** For every `π = (op_0, ..., op_{n-1})` :

```
budget(π) = Σ_{k=0}^{n-1} budget(op_k)
```

**Argument.** `budget` is defined as the component-wise triple
`(FLOPs, wall_time, energy_J)` per the `BudgetCap` dataclass.
Each component is additive over sequential execution by physical
accounting : total FLOPs = sum of FLOPs per sub-operation ; total
wall-time = sum (under the single-threaded sequential scheduler
assumed by the Dream-Episode runtime, K3) ; total energy = sum
(first-law consequence).

**Independence from [P].** Budget additivity depends only on the
resource-accounting semantics of each sub-operation, not on the
forward-callable signature. Thus (B) holds for *every*
permutation `π`, including those violating [P] — except that on
such permutations the composition itself fails closure and the
"total budget" is not observable beyond the point of failure.
This is a clean separation : the empirical failure class
compromises (C), not (B).

**Conclusion 4.** (B) holds for every `π ∈ Op`. ∎

## 5. Functional composition (proves (F))

**Claim.** For every `π` satisfying [P] and every
state `s ∈ State` :

```
φ(π)(s) = effect(π_{n-1}, ..., effect(π_0, s))
```

**Argument.** By definition of function composition in `Set`,
`(f ∘ g)(x) = f(g(x))`. Applied to the effect map :
`(effect(op_b) ∘ effect(op_a))(s) = effect(op_b, effect(op_a, s))`.
Iterating across the n-ary permutation yields the nested form
claimed in the theorem.

The statement requires [P] only indirectly : without [P], one of
the inner `effect(op_i, ·)` calls would be undefined (closure
failure, §3), and the composition would not admit a functional
interpretation. Given closure, (F) is a tautology of function
composition on `Set`.

**Conclusion 5.** (F) holds for every `π` satisfying [P]. ∎

## 6. Associativity (proves (A))

**Claim.** For every triple `(π_a, π_b, π_c) ∈ Op^3` :

```
(π_c ∘ π_b) ∘ π_a = π_c ∘ (π_b ∘ π_a)
```

**Argument.** Function composition in `Set` is associative by
mathematical definition. Explicit derivation :

```
((π_c ∘ π_b) ∘ π_a)(s) = (π_c ∘ π_b)(π_a(s))
                        = π_c(π_b(π_a(s)))

(π_c ∘ (π_b ∘ π_a))(s) = π_c((π_b ∘ π_a)(s))
                        = π_c(π_b(π_a(s)))
```

Both reduce to `π_c(π_b(π_a(s)))`. ∎

**Independence from [P].** As with (B), associativity is a
property of function composition and does not require [P]. If any
of `π_a`, `π_b`, `π_c` violates [P], the composition is
undefined ; the associativity statement becomes vacuously true on
the empty set of defined triples.

**Conclusion 6.** (A) holds on the [P]-satisfying class. ∎

## 7. Generator irreducibility of `recombine` (new in v0.2)

**Context.** Paper 1 §4.4 and §4.5 commit to the framing
"`Op` is the **free** non-commutative semigroup generated by the
four primitives". A necessary condition for the "free on four
generators" framing is that no generator is a composition of the
other three under the framework's effect semantics. v0.1 left
this open ; this section closes it for the `recombine` generator,
which is the one most likely to be objected-to by reviewers ("why
isn't `recombine` just `replay ∘ restructure` ?").

### 7.1 Claim

Let `B := {replay, downscale, restructure}`. Let
`L_B ⊆ Op` denote the subset of words whose letters are all in
`B` *and* which satisfy precondition [P] (so they are in scope of
the DR-2 theorem). Then

```
(IRRED)   ∀ ω ∈ L_B,   φ(ω) ≠ φ(recombine)
```

i.e. no composition over the other three primitives realises the
same state-transformation as the single primitive `recombine`.
`recombine` is therefore a semantically independent generator of
`Op` and is not expressible as a composition of `replay`,
`downscale`, and `restructure`.

### 7.2 Proof

We exhibit a state feature `I : State → {0, 1}^*` that is
preserved by every primitive in `B` and is *not* preserved by
`recombine`, and conclude by contrapositive.

**Invariant.** For a state `s = (W, H, M)`, let
`I(s) := sha256(H)` — a cryptographic fingerprint of the
episodic β-buffer component of `State`.

**Preservation under `B`.** For every primitive `op ∈ B` and
every state `s` :

- `op = replay` : per framework §3.1 and §4.1, `replay` samples
  from `s.H` as read-only input and applies a gradient update to
  `s.W`. It does not write to `H`. Therefore
  `effect(replay, s).H = s.H`, so `I(effect(replay, s)) = I(s)`.
- `op = downscale` : per §3.2 and §4.1, `downscale` is a
  multiplicative shrinkage of `s.W` by a factor bounded by
  invariant S1. It does not touch `H`. Therefore
  `I(effect(downscale, s)) = I(s)`.
- `op = restructure` : per §3.4 and §4.1, `restructure` mutates
  the topology of `s.W` (and correspondingly updates `s.M`). It
  does not write to `H`. Therefore
  `I(effect(restructure, s)) = I(s)`.

By induction over word length, for every `ω ∈ L_B` and every
`s ∈ State` : `I(φ(ω)(s)) = I(s)`.

**Non-preservation under `recombine`.** Per §3.5 and §4.1,
`recombine` constructs new interpolated episodes and writes them
back into `s.H` (this is the write-domain distinction between the
`B`-primitives and `recombine` : `B` writes only to `W` and `M` ;
`recombine` writes to `H`). For a non-degenerate state `s`
(i.e. a state in which the interpolation kernel produces an
episode not already in `s.H`), `effect(recombine, s).H ≠ s.H`,
hence `I(effect(recombine, s)) ≠ I(s)` with overwhelming
probability under the sha256 collision bound.

**Contrapositive.** Suppose for contradiction that there exists
`ω ∈ L_B` with `φ(ω) = φ(recombine)`. Then for every `s` :
`I(φ(ω)(s)) = I(φ(recombine)(s))`. The left-hand side equals
`I(s)` by the preservation argument ; the right-hand side
differs from `I(s)` on every non-degenerate `s` by the
non-preservation argument. Contradiction. ∎

### 7.3 Scope and limitations

- The proof is *semantic* : it establishes
  `φ(recombine) ∉ φ(L_B)` under the framework's effect semantics,
  not a purely syntactic freeness claim about the abstract
  semigroup `⟨replay, downscale, restructure, recombine⟩^+`. The
  syntactic claim (no non-trivial relations beyond associativity)
  is strictly stronger and left as future work (target G3 final,
  S8 reviewer pass) — (IRRED) is the claim that paper 1 §4.4 and
  §4.5 actually need.
- The argument relies on the framework-specific write-domain
  partition "`B` writes only to `W`/`M` ; `recombine` writes to
  `H`". A reviewer from a neuroscience background may object
  that hippocampal replay in vivo *does* modify memory traces
  (i.e. writes to the biological analogue of `H`). The response
  is that `replay` in this framework corresponds to the
  *cortical-update* phase of Walker–Stickgold consolidation —
  the phase that reads hippocampal replays and updates cortical
  weights — not the hippocampal-write phase itself. The
  framework's `replay` primitive is therefore the cortical-side
  operation, consistent with Klinzing–Niethard–Born 2019
  [@klinzing2019mechanisms] systems-consolidation framing.
- Analogous irreducibility statements for the other three
  generators (`replay`, `downscale`, `restructure` are not
  compositions of each other plus `recombine`) are expected to
  hold by similar invariants (e.g. a weights-norm monotone
  invariant distinguishes `downscale` from the others) but are
  not proved here. They are not load-bearing for paper 1 ; if a
  reviewer requests them, we add them in a v0.3 revision.

### 7.4 Corollary

The sharp four-element structure `{replay, downscale, restructure,
recombine}` is the minimal generator set for `Op` under the
effect semantics. In particular, the canonical three-process
framing of sleep consolidation [@klinzing2019mechanisms;
@rasch2025sleep] is refined, not contradicted, by our
decomposition : their "integration" process maps onto two
semantically independent primitives (`restructure` and
`recombine`), and the latter cannot be reduced to the former.

## 8. Non-commutativity landscape

`Op` is non-commutative in general. We document the most
relevant op-pair asymmetries for reviewer context ; the full
enumeration of the 16 ordered pairs is in
`docs/proofs/op-pair-analysis.md`.

### 8.1 `recombine ∘ downscale` vs `downscale ∘ recombine`

- `downscale ∘ recombine` : `recombine` samples from the
  original weight distribution, then `downscale` shrinks the
  weights post-hoc → interpolated episodes retain the generative
  diversity of the pre-shrinkage manifold.
- `recombine ∘ downscale` : `downscale` shrinks weights first →
  `recombine` then samples from a compressed manifold →
  interpolated episodes lose diversity relative to the original.

Neither ordering is "correct" ; DR-2 does not claim equivalence.
Canonical order (§4.3 of framework spec) places `downscale`
before `recombine` to preserve signal-to-noise ratio.

### 8.2 `restructure ∘ replay` vs `replay ∘ restructure`

**This is the empirically falsified pair (2026-04-21).** On the
real-weight MLX substrate with canonical input shape `(2, 4)`,
the ordering `replay ∘ restructure` (i.e. `restructure` applied
first, then `replay`) raises `ValueError` during the `REPLAY`
step because the layer swap performed by `restructure` leaves the
MLP non-callable with the canonical shape. This is the
empirical witness motivating precondition [P] (see
`docs/specs/amendments/2026-04-21-dr2-empirical-falsification.md`
§2.1 for the falsifying harness and reproduction recipe).

The reverse ordering `restructure ∘ replay` remains type-safe :
`replay` consumes through the old topology, then `restructure`
mutates it for later operations.

**Semantic asymmetry (beyond the empirical failure).** Even
setting closure aside, the two orderings would produce different
effects :
- `replay ∘ restructure` (had closure held) : consume β through
  the NEW topology (post-restructure).
- `restructure ∘ replay` : consume β through the OLD topology,
  then restructure using updated weights.

Canonical order places `replay` before `restructure` on the
serial branch (§4.3), which is consistent with [P].

### 8.3 The other 13 ordered pairs

The remaining 13 ordered pairs among the 16 total are either
trivially commutative (no pair in our set is) or asymmetric
without a closure failure. Exhaustive enumeration is in
`docs/proofs/op-pair-analysis.md` ; the summary is that among
the 24 permutations of all four primitives, exactly 12 violate
[P] and are excluded by the theorem, and the other 12 are
covered.

## 9. Fallback contract DR-2' (canonical order)

If external review reveals a gap in §7 (generator irreducibility
is the most probable locus) or in the scope caveats of §2, the
framework falls back to a strictly weaker contract :

```
DR-2' (Compositionality under canonical ordering)
∀ π = (op_0, ..., op_{n-1}) ∈ Op_canonical,
  π ∈ Op ∧ (B) ∧ (F)

where Op_canonical is the subset of permutations respecting the
canonical order
  replay < downscale < restructure < recombine
on the serial branch, supplemented with recombine as a parallel
branch.
```

DR-2' is what the P_min / P_equ / P_max runtime profiles
(`full-draft.md` §4.2) enforce by construction, so adopting it as
the primary contract does not break any current conformance
check. The trade-off is that DR-2' cannot argue
substrate-agnosticism across non-canonical orderings ; papers
would narrow the Conformance Criterion accordingly. Activation
criteria and paper-framing adjustments are tracked in
`docs/milestones/g3-decision-log.md`.

## 10. Executable harness and links to runtime

### 10.1 Executable predicate

The precondition [P] is implemented verbatim as

```python
# kiki_oniric/axioms/__init__.py : 67
def _dr2_precondition(permutation: tuple[Operation, ...]) -> bool:
    """Return True iff π satisfies weakened DR-2's precondition."""
    try:
        idx_restructure = permutation.index(Operation.RESTRUCTURE)
        idx_replay = permutation.index(Operation.REPLAY)
    except ValueError:
        return True
    return not (idx_restructure < idx_replay)
```

The function returns `True` (vacuously) for permutations that do
not contain both `RESTRUCTURE` and `REPLAY` — e.g. a
1-op permutation `(DOWNSCALE,)` or a 2-op permutation
`(REPLAY, DOWNSCALE)` trivially satisfy [P]. This matches the
existential form of [P] in §2.

The predicate is exposed on the public axioms API as
`kiki_oniric.axioms.DR2.predicate` and is tested by
`tests/conformance/axioms/test_dr2_compositionality_empirical.py`.

### 10.2 Empirical harness

The Hypothesis property test at
`tests/conformance/axioms/test_dr2_compositionality_empirical.py`
parametrises over all 24 permutations of the four primitives on
the canonical MLX substrate :

- 12 permutations (those satisfying [P]) pass closure, budget
  additivity, and effect-chaining ; the test asserts these three
  conjuncts hold.
- 12 permutations (those violating [P]) are `xfail`-parametrised
  with reason `"empirical falsification 2026-04-21 — closure
  fails on real-weight MLX substrate"`.

The test provides the empirical grounding for (C) in §3 and is
the non-trivial content of the Conformance Criterion for DR-2.

### 10.3 Conformance Criterion linkage

DR-2 is one of the three conditions of DR-3's executable
Conformance Criterion (`full-draft.md` §5, framework-C §6.2
L407–L413). A substrate `S` satisfies DR-2 iff
`test_dr2_compositionality_empirical.py` passes on `S` with 100 %
coverage of the parametrised cases. On a substrate where
`restructure` does not invalidate the forward signature, the
xfail-marked cases should be re-parametrised to pass (their
closure would no longer fail) — the test suite exposes this as a
future per-substrate parametrisation toggle.
